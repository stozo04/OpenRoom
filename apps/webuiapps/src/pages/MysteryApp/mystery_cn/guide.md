# 悬疑探案 — 用户指南

## 案件

科技公司 CEO **马库斯·维尔** 在 Lumin 产品发布晚宴当晚，被发现死于二楼上锁的书房中。当晚屋内有五位客人，凶手就在其中。发布会已取消。你有一夜时间破案。

## 玩法

你和 Kayley **合作** 调查。双方都不知道真凶——一个位于 `ws://localhost:5182` 的游戏主持人子进程持有锁定的答案。每次调查操作都会通过它处理。

### 五种操作

| 操作 | 说明 |
|------|------|
| `INTERROGATE` | 审讯嫌疑人，观察其态度标签如何在压力下变化 |
| `EXAMINE_LOCATION` | 勘察地点，右侧证据板可能出现新证据 |
| `COLLECT_EVIDENCE` | 正式将某条证据归档入案卷 |
| `READ_DOSSIER` | 打开嫌疑人的完整档案——背景、动机、时间线 |
| `MAKE_ACCUSATION` | 终局。仅一次机会。指明凶手、动机、凶器与作案方式 |

### 提示

- 态度标签只在嫌疑人在压力下 **发生变化** 时出现，这就是线索
- 某地点解锁的证据通常指向曾有机会到达那里的嫌疑人
- 准备指控时，点击右上角 **Make Accusation**。务必具体——含糊猜测必败

## 连接状态

右上角的圆点显示 GM WebSocket 状态。若为红色，说明 GM 子进程未运行——启动后会变为绿色。
